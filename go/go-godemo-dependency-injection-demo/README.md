## 常见问题解答

**Q: 为什么不用全局变量直接调用？**
A: 全局调用导致强耦合，难以测试和替换实现；DI提升可维护性和可测试性。

**Q: 是否必须使用接口？**
A: 在Go中，接口是实现DI的关键机制，它允许不同实现互换而不影响调用方。

**Q: 如何做单元测试？**
A: 可以创建 mock 实现（如 MockEmailService），在测试中注入以验证行为。

CSRF Protection Demo

什么是 CSRF 攻击

CSRF（Cross-Site Request Forgery，跨站请求伪造）是一种利用已认证用户身份发起恶意请求的攻击方式。攻击者构造恶意页面，诱骗已登录用户访问，利用用户的登录态发起非预期操作。

模拟转账（受 CSRF 保护）

此表单自动包含 CSRF Token，安全地提交转账请求。

获取 CSRF Token（API 方式）

对于 AJAX 请求，先获取 CSRF Token：

GET /api/csrf/token

Response:
{
  "headerName": "X-XSRF-TOKEN",
  "parameterName": "_csrf",
  "token": "xxxx-xxxx-xxxx-xxxx"
}

然后在请求头中携带:
X-XSRF-TOKEN: xxxx-xxxx-xxxx-xxxx

CSRF 防护原理

同步令牌模式（Synchronizer Token Pattern）：服务器生成随机 Token，嵌入到表单隐藏字段中，提交时验证 Token 是否匹配。攻击者无法获取此 Token，因此无法伪造请求。

Double Submit Cookie：将 CSRF Token 同时存储在 Cookie 和请求参数/Header 中，服务器比较两者是否一致。攻击者无法读取跨域 Cookie 值。

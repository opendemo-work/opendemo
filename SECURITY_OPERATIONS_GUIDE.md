# 🔐 安全操作指南

## 立即执行的操作

### 1. 环境设置
```bash
# 复制环境变量模板
cp .env.example .env

# 编辑配置文件，填入真实值
nano .env  # 或使用你喜欢的编辑器
```

### 2. 安全检查
```bash
# 运行安全检查
./scripts/security/check_security.sh

# 扫描敏感信息
python3 scripts/security/scan_secrets.py
```

### 3. Git 安全验证
```bash
# 检查即将提交的文件
git status

# 确保敏感文件不会被提交
git add .
git status  # 再次确认
```

## 紧急处理流程

如果发现敏感信息已经提交到 Git：

1. **立即停止推送**
2. **本地清理敏感信息**
3. **重写 Git 历史**（如有必要）
4. **更换受影响的密钥**

## 日常安全习惯

- [ ] 每次提交前运行安全检查
- [ ] 定期更新密钥和密码
- [ ] 不在代码中硬编码任何敏感信息
- [ ] 使用专业的密钥管理工具

## 联系支持

如遇紧急安全问题，请立即：
1. 停止相关服务
2. 更换所有可能泄露的密钥
3. 联系安全团队

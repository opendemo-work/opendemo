# Five-Star Documentation Standard

The five-star standard is OpenDemo's quality benchmark for all technical demonstration cases.

## Star Criteria

| Criterion | Requirement |
|-----------|-------------|
| README Coverage | ≥ 95% |
| metadata.json Coverage | ≥ 95% |
| Case Documentation Length | ≥ 4000 characters average (actual: 4500+) |
| Architecture Diagrams | Required (ASCII art format) |
| Code Examples | Complete, runnable configurations |
| Learning Key Points | Required sections |

## Case Structure

Each five-star case includes:

```
┌─────────────────────────────────────────┐
│ 1. Architecture Diagram (ASCII art)       │
│    - System architecture, data flow      │
├─────────────────────────────────────────┤
│ 2. Core Concept Explanation             │
│    - Principles, key terms, design      │
├─────────────────────────────────────────┤
│ 3. Configuration & Code Examples        │
│    - Runnable configs, complete snippets│
├─────────────────────────────────────────┤
│ 4. Command Reference Table               │
│    - Common commands, parameters        │
├─────────────────────────────────────────┤
│ 5. Learning Summary                     │
│    - Key points, best practices         │
└─────────────────────────────────────────┘
```

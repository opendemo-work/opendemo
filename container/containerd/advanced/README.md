### 1. containerd 高级特性 概述

containerd 高级特性 是容器技术栈中的重要组成部分。掌握它能够帮助你更高效地管理和运维容器化应用。


- 日常容器运维

## 🐛 常见问题

### Q1：服务启动失败？

**A**：检查容器运行时是否已安装并正常运行，查看日志定位错误。

### Q2：权限不足？

**A**：确认当前用户是否有权限操作容器运行时，必要时使用 sudo 或将用户加入相应用户组。

### Q3：命令找不到？

**A**：确认工具已正确安装，并将其加入系统 PATH。

## 🔍 containerd 命名空间详解

containerd 使用命名空间（namespace）来隔离不同用户或系统的资源。默认命名空间为 `default`，Kubernetes 使用 `k8s.io`。

```bash
# 列出命名空间
ctr namespace ls

# 在指定命名空间中查看镜像
ctr -n k8s.io images ls
```

命名空间隔离的资源包括：

- 镜像（Images）
- 容器（Containers）
- 任务（Tasks）
- 内容（Content）
- 快照（Snapshots）

在生产环境中，建议为不同环境（dev/test/prod）或不同团队分配独立命名空间。

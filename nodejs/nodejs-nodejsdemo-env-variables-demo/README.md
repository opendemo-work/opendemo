# NodeJS环境变量管理Demo

## 简介

## 常见问题解答

**Q: .env 文件是否应提交到 Git？**
A: 不应该。请将 `.env` 添加到 `.gitignore`，防止敏感信息泄露。

**Q: 如何为不同环境（开发/测试/生产）管理配置？**
A: 可使用 `.env.development`, `.env.production` 并配合 `dotenv.config({ path: '...' })` 加载对应文件。

**Q: dotenv 在生产环境中是否仍被调用？**
A: 是的，但生产中通常通过系统环境变量传入配置，`.env` 仅作为本地开发便利手段。
